# Security Policy

Demo Composer is currently alpha/unreleased.

## Reporting A Vulnerability

Do not open a public issue for vulnerability details, secrets, tokens, private URLs, or exploit steps.

Until a dedicated security contact is published, report security issues privately to the project maintainer.

Please include:

- affected version or commit
- affected component
- impact
- reproduction steps
- suggested mitigation, if known

## Security-Sensitive Areas

- authentication and session cookies
- first-run setup
- public guide links and password access
- published asset streaming
- local file storage
- Chrome extension capture and instance configuration
- CORS and deployment configuration

## Supported Versions

The project is pre-1.0 alpha. Security fixes target the current `main` branch.
