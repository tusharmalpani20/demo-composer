# Artifacts Use Editions Revisions And Publications

A Guide or Interactive Demo Artifact is stable identity only across Project Versions. Its title, description, lifecycle, and authored content belong to one Artifact Edition per Project Version; each Edition has one mutable Working Draft, immutable numbered Artifact Revisions, and independently numbered immutable Published Artifacts that identify exact Revisions.

Row Version remains concurrency metadata rather than user-visible history. Manual checkpoints, Publication, and Carry-Forward create or reuse Revisions; autosave changes only the Working Draft. Carry-Forward creates independent relational working content with new identities while reusing protected immutable media references, so later source and target edits never synchronize implicitly.
